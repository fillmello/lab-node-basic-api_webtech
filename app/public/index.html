Visão Geral

API REST construída com Node.js + Express + PostgreSQL. Suporta autenticação via JWT e controle de acesso por roles (USER, ADMIN). Esta API integra o portfólio de fillmello.

Base URL
http://localhost:3000/api
Autenticação
Authorization: Bearer <token>
Roles
USER: leitura • ADMIN: leitura + escrita

Autenticação

POST /seguranca/login
200

Gera um token JWT para usar nas demais requisições.

Body (JSON)
{
  "login": "user",
  "senha": "123"
}
Resposta (200)
{
  "id": 1,
  "login": "user",
  "nome": "User",
  "roles": "USER",
  "token": "<jwt>"
}
cURL
curl -s -X POST http://localhost:3000/api/seguranca/login \
  -H 'Content-Type: application/json' \
  -d '{"login":"user","senha":"123"}'

Produtos

Recursos protegidos: envie Authorization: Bearer <token> emitido pelo login.

GET /produtos
USER
Retorna a lista de produtos.
curl -s http://localhost:3000/api/produtos \
  -H "Authorization: Bearer <token>"
GET /produtos/:id
USER
Retorna um produto específico.
curl -s http://localhost:3000/api/produtos/1 \
  -H "Authorization: Bearer <token>"
POST /produtos
ADMIN
Cria um produto. Campos: descricao, valor, marca.
curl -s -X POST http://localhost:3000/api/produtos \
  -H 'Content-Type: application/json' \
  -H "Authorization: Bearer <token>" \
  -d '{"descricao":"Copo","valor":39.9,"marca":"Termos"}'
PUT /produtos/:id
ADMIN
Atualiza campos informados (descricao, valor, marca).
curl -s -X PUT http://localhost:3000/api/produtos/1 \
  -H 'Content-Type: application/json' \
  -H "Authorization: Bearer <token>" \
  -d '{"valor":49.9}'
DELETE /produtos/:id
ADMIN
Remove um produto.
curl -i -X DELETE http://localhost:3000/api/produtos/1 \
  -H "Authorization: Bearer <token>"

Padrão de Erros

As respostas de erro seguem o padrão:

{
  "message": "Descrição do erro"
}

Configuração

Variáveis de ambiente esperadas (arquivo .env na pasta app):

PORT=3000
DATABASE_URL=postgres://usuario:senha@host:5432/db
SECRET_KEY=uma_chave_bem_secreta
